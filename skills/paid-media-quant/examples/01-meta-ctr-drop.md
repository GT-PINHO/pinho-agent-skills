# Example 01 — Meta: CTR drop (input-side failure)

## Input (user)

Platform: Meta  
Objective: Leads  
Spend: R$ 600/day (last 7d stable)  
CPM: R$ 38 → R$ 52  
CTR (link): 1.6% → 0.8%  
CPC: R$ 1.90 → R$ 3.80  
CVR (LP): 9% (stable)  
CPA: R$ 22 → R$ 48  
Frequency: 2.1 → 3.7  
Target CPA: R$ 30

## Expected output (assistant)

**1) Dominant bottleneck**
- Bottleneck: Creative fatigue / message-market (input-side)
- Evidence: CPM up + CTR down + CPC up + frequency rising while CVR stable

**2) Diagnosis (auction)**
- CPM: rising → competition / narrowing / fatigue
- CTR: broke (1.6% → 0.8%) → hook/angle not holding
- CPC: doubled → consequence of CTR collapse
- CVR: stable → landing page likely not the first problem
- CPA: rising → downstream symptom

**3) Decision**
- Change first: creative angles + first 2 seconds / headline + audience freshness
- Do NOT change: landing page and form (CVR stable), do NOT scale budget

**4) Capital allocation plan**
- Budget action: reduce spend 20–30% to control losses while refreshing creatives
- Creative actions: test 3 new hooks for the same offer; rotate 2–3 new ads
- Audience actions: expand to fresh pockets; check exclusions; reset fatigue

**5) Validation**
- Leading indicators: CTR(link) back to ≥ 1.2%, CPM stabilization, CPC trending down
- Lagging metrics: CPA ≤ R$ 30
- Time window: 48–72h
- Kill rule: if CTR stays < 1.0% after 3 new hooks, pause the angle and replace
