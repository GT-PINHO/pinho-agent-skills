# ux-audit-fast

**Skill name:** `ux-audit-fast`

## Human alias

UX Audit Fast

## Purpose

Run a fast, high-signal audit and output:

- Top issues (prioritized)
- Proposed fix (specific)
- How to validate (what to measure / observe)

## Required inputs (minimum)

- Target page/flow (which screen, which goal)
- If possible: screenshots or URL
- Primary user action / desired outcome on that screen
- Device the audit should target (mobile / desktop / both)

## Non-negotiables

- Audit against the user’s goal, not generic “best practices”.
- Every finding has an impact note and a concrete fix — no vague advice.
- Prioritize by impact (P0/P1/P2), not aesthetics.
- If the input is missing the goal or target screen, ask before auditing.

## Checklist (what to check)

1) **Hierarchy**
- Does the headline say “what / for whom / benefit”?
- One clear primary CTA?

2) **Readability**
- Contrast, font size, spacing, line length

3) **Friction**
- Unnecessary fields/steps
- Errors and validation messages

4) **States**
- loading, empty state, error, success

5) **Consistency**
- tokens (colors/spacing/type)
- repeated components with divergent styling

6) **Accessibility baseline**
- visible focus, labels, keyboard navigation

## Output format

**Top 10 findings (priority P0/P1/P2)**
- Problem:
- Impact:
- Fix:
- Validation:
