# Example 03 — Selecting the highest-leverage lever

## Input (user)

E-commerce  
AOV: R$ 180  
Gross margin: 35%  
CAC: R$ 55  
Refunds: 2%  
Repeat purchase rate (60d): 10%  
Goal: scale from R$ 2k/day to R$ 10k/day

## Expected output (assistant)

**1) Economics snapshot**
- Contribution/order ≈ 180 * 0.35 = R$ 63
- CAC = R$ 55
- LTV assumption: low repeat; conservative LTV ≈ R$ 63–R$ 70
- LTV/CAC ≈ 1.15–1.27 (no-scale)
- Risk level: no-scale (margin too low for CAC)

**2) Dominant constraint**
- Constraint: margin is too thin for current CAC
- Evidence: contribution ~63 vs CAC 55 (almost no room for ops + volatility)

**3) Highest-leverage lever**
- Lever: margin/AOV expansion (pricing, bundles, upsells)
- Why: changing media will not create margin out of thin air

**4) Plan (executable)**
1) Increase AOV via bundles/upsell/order bump (target contribution ≥ R$ 110)
2) Improve repeat purchase via post-purchase flows (increase LTV)
3) Only then expand prospecting budget with 20% scale steps

**5) Validation**
- Leading indicators: AOV up, contribution/customer up, repeat rate up
- Lagging metrics: LTV/CAC ≥ 3, payback within tolerance
- Time window: 2–6 weeks
- Kill rule: if contribution < CAC for any segment, stop scaling that segment
