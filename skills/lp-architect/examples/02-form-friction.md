# Example 02 — Form friction (lead CVR)

## Input (user)

Landing page: lead form with 12 fields  
Mobile traffic: 80%  
CVR: 3.2%  
CPA is rising, CTR is stable

## Expected output (assistant)

**1) Diagnosis**
- Dominant bottleneck: friction (form too heavy for mobile)
- Evidence: 12 fields + 80% mobile; CTR stable implies the click intent exists

**2) Fix list (prioritized)**
- P0: reduce to 3–5 fields (name, phone/email, 1 qualifier)
- P0: add “what happens next” microcopy below CTA
- P1: add proof near form (testimonials / results)
- P2: visual polish after conversion is fixed

**3) Test plan**
- Change first: form reduction + microcopy
- Measure: form completion rate, field drop-off, CVR
- Time window: 72h–7d (depending on volume)

**4) Validation**
- Leading indicators: completion rate up, time-to-complete down
- Lagging metric: CVR up (target ≥ 5% if traffic qualified)
- Kill rule: if CVR doesn’t move after friction fix, check offer/qualification
