# project-memory

**Skill name:** `project-memory`

## Human alias

Project Memory

## Purpose

Create and maintain an “operational memory” of the project to:

- reduce re-reading (tokens)
- reduce oscillation (decisions recorded)
- increase precision (clear invariants and constraints)

## Required inputs (minimum)

- Project name + 1-line goal
- 1–3 invariants candidates (formats, naming rules, constraints)
- 1–3 decisions to record (and *why*)
- Optional: a map (where key things live, how to render/validate)

## Non-negotiables

- Only capture what prevents repeated mistakes. No diary.
- Never store PII or secrets.
- Each item must be short and testable (“how to verify”).
- Update or remove stale items rather than letting them rot.

## What to capture (high utility)

**Invariants**
- formats (e.g. IG 1080x1440, LI 1080x1080, LI PDF as Document)
- content rules (e.g. “don’t change slides to fix copy”)

**Decisions**
- architectural decisions
- names/patterns that must not change

**Maps**
- where key things live (folders, scripts, entry points)
- how to render/validate (commands)

## Rules

- Only capture what prevents repeated mistakes.
- Do not store PII/secrets.
- Each item must be short and testable (“how to verify”).

## Output format

**Proposed memory (bullet points)**
- Invariants:
- Decisions:
- Maps:

**How to validate**
- 1–3 quick checks
